Módulo 5: Vinculación con la Sociedad y Prácticas Preprofesionales
5.1 Objetivo Estratégico
Digitalizar y automatizar integralmente los procesos de prácticas preprofesionales y vinculación con la sociedad, permitiendo que los estudiantes gestionen solicitudes, documentos y aprobaciones completamente en línea, reduciendo tiempos de respuesta de 15 días a 48 horas y eliminando el uso de papel en un 90%.
5.2 Alcance del Módulo
Área	Descripción
Usuarios involucrados	Estudiantes, Tutores académicos, Coordinadores de carrera, Autoridades de facultad, Empresas/Instituciones convenio
Procesos cubiertos	Pre-registro, solicitud formal, revisión documental, aprobación por comité, ejecución de prácticas, registro de horas, evaluación y cierre, gestión de convenios, planificación de cronogramas, validación de requisitos
Documentos gestionados	Aval académico, Carta de aceptación, Informe de prácticas, Certificado de horas, Convenios institucionales, Rúbricas de evaluación, Actas de aprobación
5.3 Funciones Principales 
5.3.1 Solicitud de prácticas preprofesionales
•	Formulario multietapa con validación de requisitos previos
•	Selección de empresa/institución desde catálogo de convenios
•	Asignación automática de tutor académico según área
5.3.2 Subida de documentos institucionales
•	Almacenamiento en MongoDB GridFS o Amazon S3 (según disponibilidad)
•	Validación de formato (PDF, DOCX, JPG) y tamaño máximo (10 MB)
•	Firma digital simple con código QR de verificación
5.3.3 Seguimiento de solicitudes
•	Dashboard de estado con colores (Borrador, En revisión, Aprobada, Rechazada, Finalizada)
•	Historial de cambios y comentarios por cada paso del flujo
•	Notificaciones automáticas al cambiar de estado
5.3.4 Registro de horas realizadas
•	Registro diario con geolocalización opcional
•	Validación automática contra mínimo requerido (ej. 320 horas)
•	Reporte semanal enviado a tutor académico
5.3.5 Observaciones y correcciones
•	Espacio para que tutores o autoridades hagan comentarios
•	Control de versiones de documentos corregidos
•	Plazo máximo para correcciones (5 días hábiles)
5.3.6 Aprobación o rechazo por autoridades
•	Flujo de aprobación multinivel: Tutor → Coordinador → Decanato
•	Firma digital con timestamp
•	Generación automática de acta de aprobación en PDF
5.3.7 Notificaciones institucionales
•	Email automático con resumen de cada paso
•	Notificaciones push en el panel de control
•	Recordatorios por vencimiento de plazos
5.3.8 Gestión de empresas y convenios
•	Catálogo actualizado de empresas e instituciones con convenio activo
•	Asignación de cupos máximos por empresa por período
•	Control de vigencia de convenios y renovación automática
5.3.9 Evaluación de informes de prácticas
•	Rúbricas de evaluación configurables por carrera
•	Calificación automática basada en criterios y pesos
•	Registro histórico de evaluaciones por práctica
5.3.10 Planificación de cronogramas y tutorías
•	Creación de cronogramas por período académico y carrera
•	Agendamiento de tutorías presenciales o virtuales
•	Recordatorios automáticos de tutorías programadas
5.3.11 Validación de requisitos y cumplimiento
•	Verificación automática de créditos, materias aprobadas y documentación
•	Reglas de negocio dinámicas configurables por facultad
•	Excepciones manuales con aprobación de coordinador
5.3.12 API Gateway y BFF
•	Punto único de entrada para todos los microservicios
•	Agregación de respuestas para dashboards complejos
•	Rate limiting, autenticación centralizada y cacheo
5.4 Arquitectura de Microservicios 
5.4.1 Descripción detallada de cada servicio
Microservicio	Responsabilidad	Base de datos	Endpoints principales
internship-service	Gestión de prácticas, registro de horas, dashboard del estudiante	PostgreSQL	POST /internship, GET /internship/:id, POST /hours, GET /dashboard
document-service	Subida, validación, almacenamiento y recuperación de documentos	MongoDB	POST /document/upload, GET /document/:id, DELETE /document/:id
approval-workflow-service	Flujo de aprobaciones, firmas, actas	PostgreSQL	POST /workflow/start, PUT /workflow/approve, GET /workflow/status/:id
report-hours-service	Consolidación de horas, generación de certificados, exportación	PostgreSQL	GET /report/hours/:studentId, POST /report/certificate
internship-notification-service	Notificaciones por email y push	RabbitMQ (cola)	Consume eventos del bus
company-service	Catálogo de empresas/ instituciones con convenios activos y gestión de cupos	PostgreSQL	GET /companies, POST /companies, PUT /companies/:id, POST /companies/:id/agreement
evaluation-service	Evaluación y rúbricas de calificación de informes de prácticas	PostgreSQL	POST /evaluation, GET /evaluation/:id, PUT /evaluation/:id, GET /evaluation/internship/:internshipId
scheduling-service	Planificación de cronogramas, tutorías y fechas clave del proceso	PostgreSQL + Redis	POST /schedule, GET /schedule/:id, PUT /schedule/:id, GET /schedule/tutor/:tutorId
compliance-service	Validación automática de requisitos previos y cumplimiento normativo	MongoDB	POST /compliance/validate, GET /compliance/status/:internshipId, GET /compliance/requirements
gateway-service	API Gateway / BFF para entrada unificada, ruteo y agregación de respuestas	Redis (cache)	Rutea a todos los servicios, /api/v1/gateway/*
5.4.2 Stack tecnológico por servicio
internship-service        → NestJS + PostgreSQL + Redis (cache)
document-service          → NestJS + MongoDB + MinIO/S3
approval-workflow-service → NestJS + PostgreSQL + RabbitMQ (consumer)
report-hours-service      → Python FastAPI + PostgreSQL (pesado en cálculo)
notification-service      → NestJS + Nodemailer + RabbitMQ (producer)
company-service           → NestJS + PostgreSQL + Redis (cache)
evaluation-service        → Python FastAPI + PostgreSQL (rúbricas y cálculos)
scheduling-service        → NestJS + PostgreSQL + Redis (cache programado)
compliance-service        → Python FastAPI + MongoDB (reglas dinámicas)
gateway-service           → NestJS + Redis + express-rate-limit (BFF)
5.5 Modelo de Datos
5.5.1 Esquema PostgreSQL (internship-service)
sql
-- Tabla principal de prácticas
CREATE TABLE internship (
    id UUID PRIMARY KEY,
    student_id UUID NOT NULL,
    company_name VARCHAR(200) NOT NULL,
    company_address TEXT,
    start_date DATE NOT NULL,
    end_date DATE NOT NULL,
    total_hours_required INT DEFAULT 320,
    total_hours_completed INT DEFAULT 0,
    status VARCHAR(50) CHECK (status IN ('draft','pending_review','approved','rejected','completed')),
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- Registro de horas diarias
CREATE TABLE hour_log (
    id UUID PRIMARY KEY,
    internship_id UUID REFERENCES internship(id),
    log_date DATE NOT NULL,
    hours DECIMAL(4,2) NOT NULL,
    description TEXT,
    location_lat DECIMAL(10,8),
    location_lng DECIMAL(11,8),
    approved BOOLEAN DEFAULT FALSE
);

-- Flujo de aprobaciones
CREATE TABLE approval_flow (
    id UUID PRIMARY KEY,
    internship_id UUID REFERENCES internship(id),
    approver_role VARCHAR(50), -- tutor, coordinator, dean
    status VARCHAR(20), -- pending, approved, rejected
    comment TEXT,
    approved_at TIMESTAMP
);

5.5.3 Esquema PostgreSQL (company-service)
sql
-- Catálogo de empresas / instituciones con convenio
CREATE TABLE company (
    id UUID PRIMARY KEY,
    name VARCHAR(200) NOT NULL,
    rut VARCHAR(50) UNIQUE NOT NULL,
    address TEXT,
    phone VARCHAR(20),
    email VARCHAR(100),
    sector VARCHAR(100), -- público, privado, mixto
    status VARCHAR(20) CHECK (status IN ('active','inactive')),
    contact_name VARCHAR(100),
    contact_phone VARCHAR(20),
    max_slots INT DEFAULT 5,
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- Convenios activos con cada empresa
CREATE TABLE agreement (
    id UUID PRIMARY KEY,
    company_id UUID REFERENCES company(id),
    agreement_number VARCHAR(50) UNIQUE NOT NULL,
    start_date DATE NOT NULL,
    end_date DATE NOT NULL,
    scope TEXT,
    document_url TEXT,
    status VARCHAR(20) CHECK (status IN ('active','expired','terminated')),
    created_at TIMESTAMP DEFAULT NOW()
);

5.5.4 Esquema PostgreSQL (evaluation-service)
sql
-- Rúbrica de evaluación
CREATE TABLE rubric (
    id UUID PRIMARY KEY,
    name VARCHAR(200) NOT NULL,
    criteria JSONB NOT NULL, -- [{criterio, peso, puntaje_max}]
    min_score DECIMAL(5,2) DEFAULT 0,
    max_score DECIMAL(5,2) DEFAULT 100,
    created_at TIMESTAMP DEFAULT NOW()
);

-- Evaluación de una práctica
CREATE TABLE evaluation (
    id UUID PRIMARY KEY,
    internship_id UUID NOT NULL,
    rubric_id UUID REFERENCES rubric(id),
    evaluator_id UUID NOT NULL,
    scores JSONB NOT NULL, -- {criterio_id: puntaje}
    total_score DECIMAL(5,2),
    comments TEXT,
    status VARCHAR(20) CHECK (status IN ('draft','submitted','approved')),
    submitted_at TIMESTAMP,
    created_at TIMESTAMP DEFAULT NOW()
);

5.5.5 Esquema PostgreSQL (scheduling-service)

-- Cronograma general de prácticas
CREATE TABLE schedule (
    id UUID PRIMARY KEY,

    start_date DATE NOT NULL,

    meet_link TEXT,

    notes TEXT,
    created_at TIMESTAMP DEFAULT NOW()
);

5.7 Flujo de trabajo completo (User Journey)
Fase 1: Pre-solicitud
├── 1. Estudiante consulta convenios activos
├── 2. Selecciona empresa/institución
└── 3. Sistema verifica requisitos (materias aprobadas, créditos mínimos)

Fase 2: Solicitud formal
├── 4. Completa formulario de solicitud
├── 5. Sube documentos requeridos
└── 6. Envía a revisión → estado "En revisión"

Fase 3: Aprobación multinivel
├── 7. Tutor académico revisa y emite observaciones (si aplica)
├── 8. Coordinador de carrera valida
├── 9. Decanato da aprobación final → estado "Aprobada"
└── 10. Genera aval oficial en PDF con código QR

Fase 4: Ejecución
├── 11. Estudiante registra horas diarias
├── 12. Tutor empresa valida horas (opcional)
├── 13. Tutor académico da seguimiento quincenal
└── 14. Sistema alerta cuando se alcanza 80% de horas

Fase 5: Cierre
├── 15. Estudiante sube informe final
├── 16. Tutor académico evalúa y califica
├── 17. Sistema genera certificado de horas
├── 18. Registro automático en historial académico (vía evento Kafka)
└── 19. Notificación a titulación para continuar proceso
